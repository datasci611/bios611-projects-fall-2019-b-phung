Reading in the data

This first chunk reads in the data. We specify the col_types of the Field columns to avoid a reading warning. Off the bat, its instructive to format the date so its usable. Additionally, we extract the year and decade for each entry.

UMD_df = read_tsv("~/Documents/611/bios611-projects-fall-2019-b-phung/project_1/data/UMD_Services_Provided_20190719.tsv", col_types = cols(Field1 = col_character(), Field2 = col_character(), Field3 = col_character())) %>%
  mutate(formatted.date = as.Date(Date, "%m/%d/%Y")) %>%
  arrange(formatted.date, `Client File Number`) %>%
  mutate(year = as.numeric(format(formatted.date, "%Y"))) %>%
  mutate(decade = year - (year %% 10))

Summary by year

For a quick overview, we are interested in summarizing the data by year. In particular, we can see the numerical quantities of resources provided for each year. There were a lot of missing values in the data set, so we use the na.rm = T option. Of course, it could be that those values are non-zero, but we just dont know. Effectively, this means that this is a summary of known or recorded resources provided.

annual.summary = UMD_df %>%
  group_by(year) %>%
  summarize(count = n(),
            `Bus Tickets (Number of)` = sum(`Bus Tickets (Number of)`, na.rm = T),

            `Food Pounds` = sum(`Food Pounds`, na.rm = T),
            `Clothing Items` = sum(`Clothing Items`, na.rm = T),
            Diapers = sum(`Diapers`, na.rm = T),
            `School Kits` = sum(`School Kits`, na.rm = T),
            `Hygiene Kits` = sum(`Hygiene Kits`, na.rm = T),

Characteristics of low-frequency visitors vs.high-frequency visitors

Here, we try to investigate the differences between chronic visitors, and one- or low-time visitors. Given that chronic visitors come many times over many years, it is logically to subset the data by clients first visit date 1999 and before; it would be difficult to determine whether a visitor who started coming more recently is a chronic visitor or not. Of course, 1999 is somewhat arbitrarily chosen, and one could choose a later year to start with, but it is nice that this restriction makes the data a little more manageable.

tenure = year.filtered.UMD %>%
  left_join(visits.by.client, "Client File Number") %>%
  group_by(`Client File Number`) %>%
  summarize(visit.count = n(),
            first.visit = min(formatted.date),
            last.visit = max(formatted.date),
            time.between.first.and.last = last.visit - first.visit,
            avg.time.between.visits = time.between.first.and.last / visit.count,
            food = sum(`Food Provided for`, na.rm = T),
            n.food = sum(!is.na(`Food Provided for`)),
            nmiss.food = sum(is.na(`Food Provided for`)),
            finsup = sum(`Financial Support`, na.rm = T),
            n.finsup = sum(!is.na(`Financial Support`)),
            nmiss.finsup = sum(is.na(`Financial Support`))) %>%
  arrange(desc(time.between.first.and.last)) %>%
  filter(as.numeric(format(first.visit, "%Y")) <= 1999)

Interpreting the role of Financial Support provided (if there is one at all) is challenging given the quantity of missing values. However, we can see that even in cases where Financial Support has an abundance of known entries, many are 0. Similar things can be said of the other columns detailing resources provided.

One of the more complete columns is Food Provided for, which details the number of individuals food was provided for in a given visit. It wouldnt necessarily make sense to compare this to visit count because food provided within each visit would depend on how many are needed to be fed in the family, and collating visits by visitor would only tally up the food counts, and of course the more one visits, the more ones provided food will have aggregated. But one could do it anyway.

tenure %>%
  ggplot(aes(x = visit.count, y = food)) +
  geom_point()

Ultimately, the question we care about is what separates those who only need a little assistance to get back on their feet from those who come periodically over the course of decades, because we would like to move people from the latter group to the former. UMD provides the same support to those in need indiscriminately as far as can be discerned from this data, so it stands to reason that the issue is a systemic one far broader than we in the micro are able to address. This is not to criticize; in fact, UMDs services are very admirable, and undoubtedly improve peoples lives. Rather, food and shelter, while necessary, are not sufficient in themselves to sustain someone in need.
